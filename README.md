# Система мониторинга вакансий

Веб-приложение для автоматического мониторинга вакансий различных специальностей, управления источниками данных, настройки правил поиска и просмотра аналитики по найденным объявлениям.

## Стек

- Frontend: `Vite + TypeScript + HTML + CSS + Chart.js`
- Backend: `Node.js + Express + TypeScript`
- Хранение данных: `SQL.js` с локальным сохранением SQLite-файла
- Тесты: `Vitest + Supertest`

## Возможности

- панель мониторинга со сводными показателями и журналом событий;
- каталог вакансий с фильтрацией, таблицей и карточкой изменений;
- управление правилами мониторинга и источниками данных;
- автоматический запуск мониторинга по активным cron-правилам;
- сохранение тематических подборок и повторная работа с отобранными позициями;
- аналитика по динамике вакансий, компаниям и качеству источников;
- ручной запуск цикла мониторинга;
- локальное хранение данных без внешних сервисов.

## Запуск

```bash
npm install
npm run dev
```

Клиентская часть работает через Vite, сервер API запускается параллельно.

После старта система автоматически создаёт начальные демонстрационные данные, журнал событий и первую рабочую подборку.

## Проверка

```bash
npm run check
```

## Production

```bash
npm run build
npm start
```

По умолчанию production-сервер доступен по адресу `http://localhost:4173/`.

## Vercel

Проект подготовлен для бесплатного деплоя на Vercel: статическая часть собирается в `dist`, а Express API запускается через serverless-функцию `api/index.ts`.

На Vercel SQLite-файл создаётся во временном каталоге `/tmp`, поэтому приложение запускается без внешней базы данных, но сохранённые изменения могут сбрасываться после холодного старта функции. Для постоянного хранения понадобится внешняя БД или read-only интеграция с внешним API вакансий.

## Разделы интерфейса

- `/` — оперативная сводка по состоянию контура, последним изменениям и журналу;
- `/vacancies` — каталог вакансий с фильтрами, историей изменений и сохранением в подборки;
- `/rules` — правила мониторинга и переключение активности расписаний;
- `/sources` — источники данных и их техническое состояние;
- `/analytics` — графики, распределения по регионам и диапазонам оплаты;
- `/logs` — журнал системных и операторских событий;
- `/collections` — сохранённые подборки вакансий.
